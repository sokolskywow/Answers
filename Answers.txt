1. Задача по написанию тест-кейсов:
	Условие: Разработайте тест-кейсы для формы логина, которая содержит два поля ввода: "Username" и "Password", и кнопку "Login". Условия:
		Поле "Username" должно быть обязательным и должно принимать только email-адреса.
		Поле "Password" должно быть обязательным и должно содержать не менее 8 символов.
		Кнопка "Login" должна быть активна только при корректном заполнении обоих полей.
	Ожидаемый результат: Описание позитивных и негативных тест-кейсов.
	
	Кейсы:
		Поле "Username" должно быть обязательным и должно принимать только email-адреса.
			1. Позитивный
			Предусловие: поле "Password" корректно заполнено (>= 8 символов)
				Заполнить поле "Username" корректным значением (проверка по маске email-адреса https://en.wikipedia.org/wiki/Email_address#Valid_email_addresses)
			Промежуточный результат: Кнопка "Login" активна
				Нажать кнопку "Login"
			Ожидаемый результат: успешная авторизация 
			2. Негативные - альтернативные сценари позитивного кейса
				2.1 - "поле Username не заполнено" - кнопка Login неактивна, возможно отображение информационного уведомления под полем Username в рамках постраничной валидации (см. спецификацию)
				2.2 - "введённое значение Username не соответствует формату" - кнопка Login неактивна, возможно отображение информационного уведомления под полем Username в рамках постраничной валидации (см. спецификацию)
		Поле "Password" должно быть обязательным и должно содержать не менее 8 символов.
			3. Позитивный
			Предусловие: поле Username корректно заполнено
				Заполнить поле Password корректным значением (>= 8 символов)
			Промежуточный результат: Кнопка "Login" активна
				Нажать кнопку "Login"
			Ожидаемый результат: успешная авторизация
			4. Негативные - альтернативные сценари позитивного кейса
				4.1 - "поле Password не заполнено" - кнопка Login неактивна, возможно отображение информационного уведомления под полем Password в рамках постраничной валидации (см. спецификацию)
				4.2 - "введённое значение Password не соответствует формату (<8 символов)" - кнопка Login неактивна, возможно отображение информационного уведомления под полем Password в рамках постраничной валидации (см. спецификацию)
		Кнопка "Login" должна быть активна только при корректном заполнении обоих полей. (косвенно проверяется в рамках предыдущих кейсов)
			5. Позитивный
				Заполнить поле "Username" корректным значением
				Заполнить поле Password корректным значением
			Ожидаемый результат: кнопка Login активна
			6. Негативный
				6.1 - "Корректный Username, некорректный Password" - Кнопка Login неактивна
				6.2 - "Корректный Password, некорректный Username" - Кнопка Login неактивна
				6.3 - "Некорректный Username, некорректный Password" - Кнопка Login неактивна

3. Задача по теории вероятности:
Условие: Монету подбрасывают три раза. Какова вероятность того, что выпадет ровно два орла?
Ожидаемый результат: Описание решения и ответ
	
	все возможные варианты выпадения (8)
		РРР, РРО, РОР, РОО
		ОРР, ОРО, ООР, ООО
	подходящие под условие (3)
		РОО, ОРО, ООР
	итого: 3/8 = 0,375
